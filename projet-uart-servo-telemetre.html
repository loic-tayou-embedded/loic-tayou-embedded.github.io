Chaîne UART – Servomoteur – Télémètre sur FPGA

Type : Projet personnel – Polytech Paris-Saclay · Août 2025 – Septembre 2025 · 2 mois

VHDL UART RX/TX PWM servomoteur Télémètre ultrason ModelSim Testbenches & scripts .do

Résumé

L’objectif du projet est de concevoir, en VHDL sur FPGA, un système intégrant plusieurs IP matérielles : un module implémentant le protocole UART pour l’échange avec un terminal série, un module pour le pilotage d’un servomoteur, ainsi qu’un module pour le pilotage d’un télémètre ultrason. L’ensemble est piloté par un automate de contrôle qui sélectionne le mode de fonctionnement via les switches SW1 et SW0 :

  • SW1 SW0 = 00 : aucun IP (UART, servomoteur, télémètre) n’est actif.
  • SW1 SW0 = 01 : commande du servomoteur depuis le PC (via le bus UART).
  • SW1 SW0 = 10 : envoi cyclique des mesures de distance vers le PC via le bus UART.
  • SW1 SW0 = 11 : commande automatique du servomoteur à partir des distances mesurées par le télémètre.

Tâches accomplies

  • Mettre en œuvre le protocole UART en VHDL.
  • Développer en VHDL un module paramétrable générant des signaux PWM, puis un module permettant de piloter le servomoteur à partir de ce générateur, via une sortie GPIO de la carte.
  • Développer en VHDL un module permettant de commander le télémètre ultrason pour réaliser des mesures de distances.
  • Implémenter l’automate permettant de sélectionner un mode de fonctionnement via les switches SW0 et SW1.
  • Valider chaque IP par simulation fonctionnelle : testbenches + scripts Tcl (.do).

Environnement technique

ModelSim, Quartus, PuTTY, VHDL, Oscilloscope, Breadboard, FPGA (carte DE1), Servomoteur, Télémètre ultrason, Cycle en V
