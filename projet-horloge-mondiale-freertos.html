Horloge mondiale avec affichage déporté sous FreeRTOS

Type : Projet personnel – DE1 + STM32 Nucleo + ESP32 – FreeRTOS

Nios II FreeRTOS SPI CAN I2C UART STM32 ESP32

Résumé

Ce projet consiste en un système d’affichage temps réel qui montre l’heure de différentes villes et la température ambiante sur un écran OLED. Le système est divisé en deux parties interconnectées : un module principal comprenant un processeur Nios II (sous FreeRTOS) sur la carte DE1, couplé à une carte STM32 Nucleo interfacée avec un capteur de température ; un module d’affichage comprenant un ESP32 (sous FreeRTOS) pilotant un écran OLED. Au démarrage, le Nios II demande l’heure de Paris via le terminal série, puis affiche en continu la ville et l’heure actuelles tout en permettant à l’utilisateur de changer de ville et de régler l’heure. Ensuite, chaque seconde, le Nios II transmet la ville et l’heure à la STM32 via le bus SPI, qui relaie ces informations ainsi que la température mesurée à l’ESP via le bus CAN pour l’affichage sur l’écran OLED.

Tâches accomplies

  • Architecturer un système autour du processeur softcore Nios II, intégrant des IP core interfacés via le bus Avalon.
  • Implémenter un driver UART en polling pour les accès en lecture et écriture sur le terminal série.
  • Concevoir une tâche centralisant l’accès à la ressource partagée qu’est la console pour éviter les conflits d’accès et garantir un affichage stable sur le terminal série.
  • Réaliser la synchronisation inter-tâches via le mécanisme des queues FreeRTOS.
  • Mettre en œuvre un timer matériel avec interruptions pour réaliser le décompte temporel.
  • Implémenter une tâche dédiée avec accès exclusif au bus SPI.
  • Implémenter une queue de données pour le transfert des données (ville + heure) à la tâche pilotant le bus SPI.
  • Réceptionner les données provenant de la carte DE1 via le bus SPI et lire les données de température via le bus I2C sur la carte STM32.
  • Transmettre via le bus CAN les données de ville et d’heure chaque seconde, et les données de température chaque minute.
  • Réceptionner sur la carte ESP32 les données transmises via le bus CAN, puis mettre à jour l’affichage sur l’écran OLED via le bus I2C (ville, heure, température).

Environnement technique

STM32CubeIDE, Quartus, Arduino IDE, Nios II Software Build Tools for Eclipse, PuTTY, Tera Term, VHDL, Co-design matériel/logiciel, C, CAN, I2C, SPI, UART
