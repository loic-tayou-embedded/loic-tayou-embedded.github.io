Reverse engineering d’un robot aspirateur Ultenic D5S Pro

Type : Projet de groupe – 5ᵉ année cycle ingénieur

Projet de groupe Reverse engineering MCU GD32F103VET6 C bas niveau Robot aspirateur

Résumé du projet

Projet de 5ᵉ année autour d’un robot aspirateur Ultenic D5S Pro : analyse de l’architecture matérielle (moteurs, capteurs, électronique de commande) et ajout d’un LiDAR pour améliorer la navigation. Un modèle du robot a été recréé sous Webots par d’autres membres de l’équipe pour tester les stratégies de déplacement et les interactions capteurs/actionneurs.

Ce que j’ai réalisé

  • Exploration du PCB principal du robot et identification du microcontrôleur GD32F103VET6 en boîtier LQFP100.
  • Étude approfondie de la documentation constructeur pour :
    • identifier les broches de programmation (SWD / UART) et les modes de boot,
    • connaître les niveaux de tension et conditions de démarrage,
    • localiser les broches utiles sur le PCB réel.
  • Déprotéger la mémoire flash via le logiciel STM32 ST-LINK Utility. Pour flasher la carte, j'ai utilisé le programmateur (Débogueur/programmeur) ST-LINK/V2.
  • Mise en place de la chaîne de programmation du microcontrôleur (connexion physique, tensions, configuration du mode).
  • Développement de codes C bas niveau pour communiquer avec les capteurs et actionneurs de la carte (lecture de capteurs, commande moteurs, etc.).
  • Utilisation du robot virtuel sous Webots (créé par l’équipe) comme banc de test pour mes algorithmes avant essais sur robot réel.

Techno & outils

Microcontrôleur GD32F103VET6, C embarqué bas niveau, SWD/UART, oscilloscope, reverse engineering PCB, Webots (simulation).
